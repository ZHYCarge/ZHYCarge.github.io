解决代码冲突小记

发表于 2025-06-30 分类于 学习 阅读次数： Waline： 本文字数： 726 阅读时长 ≈ 3 分钟

前情摘要

最近在编写工作用偷懒小代码，仓库项目存在两个分支， dev 与 master 分支，由于无聊且想要熟悉下Git的使用技巧，因此对 master 分支设置了分支保护，至此，本篇水文出现。

详细描述……

分支保护的愁与泪

由于开启了分支保护，所以通过使用Git命令无法直接push到 master 分支，因此，所有的更改都需要通过先推送到 dev 分支，由 dev 分支发起PR请求后才能将代码合并到 master 分支，但是常规PR解决都是最终要推送master分支上的，（可能是我孤陋寡闻，或者Gitee是这么推荐的。。。），由于分支保护，因此此条路径不通，遂问DeepSeek。

DeepSeek问询语

解决方法

  1. 拉取最新代码到本地
  2. 将master合并到dev分支（解决冲突的核心步骤） 此时会出现冲突提示（类似 CONFLICT (content): Merge conflict in <文件名>）
  3. 解决冲突
    选择保留需要的代码（或合并两者），删除 <<<<<<</=======/>>>>>>> 标记
    对每个冲突文件重复此操作
  4. 提交合并结果并推送远端分支 这会自动更新已有的PR（PR会实时跟踪dev分支的新提交）
  5. 完成PR合并请求
    之后便可在平台上对已有PR进行后续操作

继续问询
